YeoboSay ver0.2 앱 내부 전화 수신 기획서

작성일: 2026-05-27 / 문서 범위: Backend + Android + API Contract

ver0.2 앱 내부 전화 수신 WebSocket MVP 확장

ver0.2의 목표는 실제 전화망을 사용하지 않고, 백엔드가 Android 앱에 "전화가 걸려오는 것처럼 보이는" 이벤트를 전달하는 것이다. 사용자는 Android 앱 안에서 수신 화면을 보고 전화를 받거나 거절할 수 있으며, 수락하면 기존 ver0.1 AI 음성 대화 흐름으로 진입한다.

1. 목적

ver0.1은 사용자가 직접 통화 시작 버튼을 눌러 AI 음성 대화를 시작하는 구조였다. ver0.2에서는 반대로 백엔드가 Android 앱에 통화 요청 이벤트를 보내고, Android가 앱 내부 수신 화면을 띄우는 흐름을 추가한다.

이 기능은 향후 보호자 예약 전화, AI 안부 전화, 푸시 알림 기반 백그라운드 수신, WebRTC 통화로 확장하기 위한 중간 단계다.

2. 핵심 결정

  • 실제 전화번호 기반 통화는 ver0.2에서도 제외한다.
  • Android 앱이 켜져 있는 상태에서만 수신 이벤트를 받는 것으로 시작한다.
  • Backend → Android 이벤트 전달은 WebSocket을 사용한다.
  • 테스트 편의를 위해 Android 앱 내부에 "테스트 전화 요청" 버튼을 추가한다.
  • 전화 수락 후에는 기존 POST /call-sessions 기반 음성 대화 흐름을 재사용한다.
  • FCM, full-screen notification, 백그라운드 수신은 ver0.2 이후로 분리한다.

3. MVP 범위

포함하는 기능

  • Android 앱 실행 중 WebSocket 연결
  • Android 테스트 버튼을 통한 전화 요청 트리거
  • Backend call invitation 생성
  • Backend에서 Android로 incoming call 이벤트 발행
  • Android 앱 내부 수신 화면 표시
  • 수락 / 거절 액션
  • 수락 시 기존 AI 음성 대화 세션 시작
  • 거절 시 Backend에 거절 상태 전달
  • 기본 서버 로그와 앱 상태 표시

포함하지 않는 기능

  • 실제 전화번호 발신
  • Twilio 등 외부 전화망 연동
  • 앱 백그라운드 상태에서의 수신
  • FCM 푸시 알림
  • Android full-screen incoming call notification
  • WebRTC media stream
  • OpenAI Realtime API 직접 연동
  • 사용자 인증 기반 특정 기기 라우팅

4. 전체 동작 흐름

Android 앱 실행
→ Android가 Backend WebSocket에 연결
→ Android에서 테스트 전화 요청 버튼 클릭
→ Android가 Backend에 POST /call-invitations/test 요청
→ Backend가 call invitation 생성
→ Backend가 WebSocket으로 incoming_call 이벤트 발행
→ Android가 앱 내부 전화 수신 화면 표시
→ 사용자가 받기 또는 거절 선택
→ 받기: Android가 POST /call-invitations/:id/accept 요청
→ 받기: Android가 기존 POST /call-sessions 요청
→ 기존 ver0.1 AI 음성 대화 화면으로 진입
→ 거절: Android가 POST /call-invitations/:id/decline 요청
→ Backend가 invitation 상태를 ACCEPTED 또는 DECLINED로 기록

6. API 초안

테스트 전화 요청

POST /call-invitations/test

Android 앱 내부 테스트 버튼에서 호출한다.

{
  "id": "call-invitation-id",
  "status": "RINGING",
  "callerName": "YeoboSay",
  "message": "AI 말벗 전화가 왔어요.",
  "createdAt": "2026-05-27T10:00:00.000Z",
  "expiresAt": "2026-05-27T10:00:30.000Z"
}

전화 수락

POST /call-invitations/:id/accept

수락 후 Android는 기존 POST /call-sessions를 호출해 음성 대화 세션을 시작한다.

전화 거절

POST /call-invitations/:id/decline

WebSocket 이벤트

{
  "type": "incoming_call",
  "callInvitationId": "call-invitation-id",
  "callerName": "YeoboSay",
  "message": "AI 말벗 전화가 왔어요.",
  "createdAt": "2026-05-27T10:00:00.000Z",
  "expiresAt": "2026-05-27T10:00:30.000Z"
}

7. 데이터 모델 초안

enum CallInvitationStatus {
  RINGING
  ACCEPTED
  DECLINED
  EXPIRED
}

model CallInvitation {
  id        String               @id @default(cuid())
  status    CallInvitationStatus @default(RINGING)
  callerName String
  message   String
  createdAt DateTime             @default(now())
  expiresAt DateTime
  acceptedAt DateTime?
  declinedAt DateTime?
}

초기 구현에서 DB 저장이 부담되면 메모리 기반으로 시작할 수 있다. 다만 시연 이후 디버깅과 상태 추적을 생각하면 DB 저장을 권장한다.

8. Android 화면 상태

상태 설명 주요 UI
Disconnected WebSocket 연결 전 또는 끊김 연결 상태 표시, 재연결 시도
Connected 전화 이벤트 수신 가능 테스트 전화 요청 버튼
Ringing incoming_call 이벤트 수신 수신 화면, 받기, 거절
Accepting 수락 요청 처리 중 버튼 비활성화, 진행 표시
Declining 거절 요청 처리 중 버튼 비활성화, 진행 표시
InCall 기존 AI 음성 대화 화면 진입 ver0.1 CallScreen 재사용

10. 테스트 계획

  • Backend: invitation 생성, 수락, 거절, 만료 단위 테스트
  • Backend: WebSocket gateway가 incoming_call 이벤트를 발행하는지 테스트
  • Android: WebSocket 연결 성공/실패 상태 표시 확인
  • Android: 테스트 버튼 클릭 시 수신 화면이 표시되는지 확인
  • Android: 받기 선택 시 기존 AI 음성 대화 화면으로 진입하는지 확인
  • Android: 거절 선택 시 수신 화면이 닫히는지 확인
  • 수동 테스트: Backend 실행, Android Emulator 실행, 전체 흐름 녹화

11. 이후 확장

  • FCM 기반 백그라운드 전화 수신
  • Android full-screen notification
  • 사용자/기기 식별 기반 특정 기기 라우팅

12. 완료 기준

  • Android 앱 실행 중 Backend WebSocket에 연결된다.
  • Android 테스트 버튼으로 전화 요청을 만들 수 있다.
  • Backend가 incoming_call 이벤트를 Android로 보낸다.
  • Android가 앱 내부 수신 화면을 표시한다.
  • 사용자가 받기를 누르면 기존 AI 음성 대화가 시작된다.
  • 사용자가 거절을 누르면 invitation이 거절 상태로 처리된다.
  • Backend와 Android 변경은 각각 별도 PR로 나눌 수 있다.
